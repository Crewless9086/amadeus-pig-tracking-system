# Gold Standard Examples

## Gold Standard Mission Review

A good CHARLIE review packet says:

- mission: exact id and title;
- owner request: one-paragraph summary;
- outcome: what is now true;
- changed files: exact paths;
- evidence: commands run and pass/fail results;
- UI proof: screenshot/live preview when relevant;
- risks: what could still fail;
- release state: PR/commit/deploy status;
- owner action: approve final, send back, pause, reject, or mark done.

Bad review packet:

- says "done" without tests;
- uses old screenshots;
- hides approval buttons;
- does not explain risks;
- does not show changed files;
- makes owner chase technical artifacts.

## Gold Standard UI Control

For owner review cards:

- direct buttons visible on the card;
- comments field close to the decision;
- return stage visible when sending back;
- full review available but not required for basic decision;
- generated screenshot clearly labelled if not live evidence.

## Gold Standard Agent Charter

Each agent charter must include:

- role;
- watches;
- inputs;
- outputs;
- can suggest;
- can prepare;
- cannot do;
- owner gates;
- source-of-truth data;
- dashboard placement;
- review evidence.

## Gold Standard Business Build

For a new income stream:

- business purpose is clear;
- customer path is mapped;
- pricing/payment rules exist;
- operations and fulfilment are known;
- legal/privacy gaps are recorded;
- source-of-truth data exists;
- agent roles are defined;
- launch evidence and owner gates are defined.

## Gold Standard Dashboard

An owner dashboard should answer in the first view:

- What needs my attention?
- What is active?
- What is blocked?
- What can I approve?
- What is the system not allowed to do?
- What evidence supports this?

## Gold Standard Data Change

A data change or migration should show:

- source data;
- target table;
- dry-run result;
- conflicts/quarantine;
- migration file;
- rollback plan;
- verification query/result;
- affected docs updated.
