# Dashboard UI Playbook

Use `../07-standards/UI_DASHBOARD_STANDARD.md`.

## Required Evidence

- actual screen, local preview, or live preview;
- critical buttons visible where the decision appears;
- desktop/mobile check when practical;
- text does not overflow;
- cards do not hide owner decisions;
- source data and loaded state are clear;
- empty, loading, error, blocked, and unavailable states are explicit;
- actions give feedback.

## CHARLIE Dashboard Checks

For `/charlie`, verify:

- mission counts;
- active runner state;
- next approved mission;
- review backlog;
- review decision buttons;
- command center truth;
- Vault/data/model/tool readiness;
- runner boundary: dashboard does not run shell commands.
